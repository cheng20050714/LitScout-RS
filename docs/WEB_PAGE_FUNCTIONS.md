# LitScout-RS 页面功能汇总


Web 工作台分为三块：配置页、调研页、阅读库页。常见使用顺序是：先配置模型和令牌，再创建调研任务，最后把需要细读的 arXiv 论文加入阅读库。

---

## 01 配置页

配置页是 **连接准备型** 页面，用来填写当前浏览器会话需要的服务信息。

- **模型配置**：填写模型 API Key、DeepSeek 兼容接口地址、报告主模型和轻量模型。
- **访问配置**：填写 GitHub Token，提高 GitHub API 调用额度。
- **健康检查**：查看本机后端、LLM、GitHub Token 是否可用。
- **会话保存**：配置只保存在浏览器会话里，不写入项目文件、trace 或 checkpoint。

如果后端没有启用 LLM，页面里也没有填写模型 API Key，系统不会进入调研工作台。

---

## 02 调研页

调研页是 **任务执行型** 页面，负责一次完整 research run：生成计划、抓取 GitHub 和 arXiv 资料、整理证据、检查覆盖度、生成报告和审计引用。

### 左侧任务控制

任务控制区是 **任务创建型** 功能。

- **主题输入**：输入中文调研主题，也可以点击示例主题快速填充。
- **抓取预算**：设置 GitHub 仓库数量、arXiv 论文数量和章节数量上限。
- **流程开关**：可以跳过计划质量检查或证据覆盖检查。
- **任务创建**：创建 stateful run 后，系统会生成调研摘要、章节计划和查询组合。

### 计划标签

计划标签是 **计划确认型** 功能，用来在正式抓取前检查和修改系统生成的计划。

- **计划预览**：查看调研摘要、章节结构、GitHub 查询词、arXiv 查询词和规划理由。
- **人工修订**：修改章节标题、研究问题、查询词、规划理由和修订备注。
- **流程推进**：保存修订，或批准计划并开始抓取。

只有 run 处于“计划待确认”时才能编辑。任务开始后，计划会锁定。

### 运行状态抽屉

运行状态抽屉是 **过程诊断型** 功能，显示当前 run 跑到哪一步。

- **进度查看**：显示整体进度、当前状态、后端状态和模型状态。
- **时间线查看**：展示从计划、抓取、证据整理、报告生成到完成或失败的状态变化。
- **checkpoint 管理**：查看 checkpoint，并在允许的位置新建分支继续跑。
- **事件排查**：查看最近事件和报告输出路径，方便定位卡住或失败的位置。

### 证据标签

证据标签是 **来源核对型** 功能，用来查看本次调研抓到的 GitHub 和 arXiv 资料。

- **证据浏览**：按章节展示来源类型、引用 ID、标题、链接、中文说明、原始片段和查询词。
- **查询记录**：查看失败查询和完整查询历史。
- **加入阅读库**：arXiv 证据可以一键加入阅读库；GitHub 项目不会进入阅读库。

### 覆盖标签

覆盖标签是 **质量检查型** 功能，用来判断每个章节的证据是否足够。

- **覆盖概览**：显示整体覆盖度和系统建议。
- **章节缺口**：查看每章证据数量、缺口严重程度和缺口解释。
- **补查建议**：在证据不足时给出推荐补充查询词。

这个页面适合在生成报告前查看。如果某章证据明显偏少，可以回到计划或从 checkpoint 新建分支补查。

### 引用标签

引用标签是 **引用审计型** 功能，用来检查报告里的引用风险。

- **引用覆盖**：查看段落引用覆盖率和来源多样性。
- **URL 检查**：确认报告引用是否在允许来源范围内。
- **问题提醒**：标出时效性警告、缺引用段落和外部 URL 违规。

它只指出问题，不直接改报告。

### 报告标签

报告标签是 **结果阅读型** 功能，用来查看最终 Markdown 报告和继续追问。

- **报告预览**：调研未完成时显示摘要、章节计划、查询词和 run 状态。
- **报告阅读**：调研完成后渲染 Markdown 报告。
- **中文翻译**：可以用模型翻译报告，并检查原始引用 URL 是否保留。
- **报告追问**：围绕整次调研继续提问，回答会流式显示。

报告页适合问整体问题，比如哪些项目值得先读、哪些证据最薄弱、下一轮该补哪些查询。

---

## 03 阅读库页

阅读库页是 **论文深读型** 页面，用来保存 arXiv 论文，并生成基于全文或近全文的阅读笔记。

### 论文列表

左侧列表是 **论文管理型** 功能。

- **论文浏览**：查看已加入论文的标题、处理状态和文本覆盖类型。
- **状态识别**：区分待生成、抓取中、提取中、已完成、全文获取失败和笔记生成失败。
- **快速切换**：点击论文后，在右侧打开详情。

### 论文详情

右侧详情是 **单篇操作型** 功能。

- **原文入口**：打开 arXiv 原文页面。
- **笔记生成**：生成或重新生成阅读笔记。
- **论文删除**：从阅读库移除论文，同时清理对应缓存。
- **错误查看**：查看全文获取或笔记生成失败原因。
- **旧笔记提示**：旧版本或摘要级笔记会标记为质量未知或不可靠。

### 全文获取诊断

全文获取诊断是 **抓取解释型** 功能，用来说明笔记为什么成功或失败。

- **来源信息**：显示文本来源、提取器、字符数、页数和质量分。
- **失败尝试**：显示最近一次失败 attempt，方便判断卡在哪个步骤。
- **链路说明**：Jina PDF 失败不一定代表最终失败，系统还会尝试直接下载 arXiv PDF，并用本地 `pdf-extract` 提取文本。

### 阅读笔记和单篇追问

阅读笔记是 **全文分析型** 功能，只在全文或近全文通过质量门控后生成。

- **笔记内容**：包含速览、问题、贡献、方法、实验、局限、复现要点和后续追问建议。
- **失败保护**：全文获取失败时，不会生成摘要级长文，而是显示失败说明。
- **论文问答**：可以围绕当前论文继续提问，聊天历史会随论文恢复。

---

## 页面之间的关系

```text
配置页
  -> 调研页创建任务
  -> 计划标签确认查询
  -> 证据标签查看来源
  -> 把 arXiv 论文加入阅读库
  -> 阅读库生成全文笔记
  -> 围绕报告或单篇论文追问
```

- **报告页** 面向整次调研，回答“这个方向整体有什么结论”。
- **阅读库页** 面向单篇论文，回答“这篇论文具体讲了什么”。

---

## 当前边界

- 阅读库只接收 arXiv 论文，不接收 GitHub 项目。
- 详细阅读笔记必须基于通过质量门控的全文或近全文。
- Jina 失败后，系统会继续尝试直接 PDF 下载和本地提取。
- GitHub Token 可选；不配置也能跑，但会使用未认证 API 额度。
